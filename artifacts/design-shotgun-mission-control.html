Design Shotgun Fallback

Three calm mission-control directions for Auto-Research.

The native design binary is unavailable here, so this comparison board uses repo-grounded HTML mockups. All three variants honor the same backend contract and operator goals, but they differ in how strongly they center timeline, fleet control, and execution context.

Shared read model: /api/dashboard Shared action model: /api/workbench/control Design tone: calm, analytical, non-flashy
