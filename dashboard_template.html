Auto Research Trading Workbench

One launcher, one UI, ten isolated experiments.

The dashboard now supervises the paper feed plus a message-based experiment manager. Each thread gets its own state, lifecycle, and controls without sharing writable research state with the others.

Autostart is on by default. Launching the workbench starts the UI, the paper feed, and the experiment manager together unless you disable it with WORKBENCH_AUTOSTART=0.
Paper Feed Control

Managed process for the Jupiter paper-mode loop.

Guarded: start locks out while running
Start relaunches a stopped feed. Restart is for a clean relaunch of an active feed. Stop halts the managed loop.
uv run python workbench_ctl.py status
uv run python workbench_ctl.py restart-paper
Experiment Manager

Message-backed lifecycle manager for the 10 isolated Jupiter research threads.

Guarded: manager state fans out to threads
Start launches the experiment manager, restart recreates the manager process, and stop halts all experiment orchestration.
uv run python workbench_ctl.py start-manager
uv run python workbench_ctl.py stop-manager
